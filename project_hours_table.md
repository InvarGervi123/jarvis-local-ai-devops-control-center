# פירוט שעות עבודה - פרויקט גמר J.A.R.V.I.S (סך הכל: 300 שעות)

הטבלה הבאה מפרטת את חלוקת הזמנים ההגיונית והריאלית לפיתוח פרויקט בסדר גודל כזה (Full-Stack SaaS + Chrome Extension + AI), והיא מותאמת להגשה אקדמית או לפרויקט גמר.

| שלב בפרויקט | תיאור המשימות שבוצעו | שעות עבודה |
| :--- | :--- | :--- |
| **1. מחקר, אפיון ותכנון (Research & Planning)** | אפיון דרישות המערכת, תכנון ארכיטקטורת Client-Server, למידת ה-API של Google Gemini, תכנון סכמות למסד הנתונים, ויצירת סקיצות UI/UX. | 35 שעות |
| **2. הקמת סביבת הפיתוח והתשתיות (Setup)** | הגדרת סביבת עבודה עם Vite/React, הקמת שרת Node.js/Express, חיבור והגדרת מסד נתונים בענן (MongoDB Atlas), והגדרת משתני סביבה (ENV). | 15 שעות |
| **3. פיתוח צד שרת - הרשאות ו-API (Backend Core)** | כתיבת ה-REST API, יצירת מודלים ב-Mongoose (משתמשים, היסטוריה), פיתוח מערכת הרשמה/התחברות, הצפנת סיסמאות (Bcrypt), וניהול חותמות אבטחה (JWT Middleware). | 45 שעות |
| **4. פיתוח צד לקוח - ממשק והרשאות (Frontend Core)** | בניית ארכיטקטורת React, ניתוב דפים (React Router), ניהול State גלובלי עם Context API, עיצוב דפי Login/Register, פיתוח ה-Dashboard עם תצוגת נתונים וגרפים (Recharts). | 50 שעות |
| **5. שילוב בינה מלאכותית (AI Integration)** | חיבור מודל Gemini לשרת, פיתוח ארכיטקטורת אבטחה BYOK להעברת מפתחות API, כתיבת פרומפטים מורכבים (JSON parsing), וטיפול בניתוח תמונות (Multimodal Vision API). | 45 שעות |
| **6. פיתוח מסכי האפליקציה (AI Modules UI)** | עיצוב ופיתוח מורכב של מסכי המערכת: Document Review (עורך טקסט מפוצל), Screen Analysis (העלאת תמונות וניתוח), Admin Panel (משיכת נתוני שרת CPU/RAM חיים), ומסך הקלטות. | 40 שעות |
| **7. פיתוח תוסף הדפדפן (Chrome Extension)** | למידה וכתיבה של Manifest V3, פיתוח Background Service Worker, הזרקת Content Scripts לאתרים, פיתוח מנגנון סנכרון Token אוטומטי בין האתר לתוסף, וקריאות API מהדפדפן. | 35 שעות |
| **8. בדיקות איכות, דיבוג ושיפורים (Testing & QA)** | בדיקות מקיפות לשרת ב-Postman, איתור ותיקון באגים בסנכרון ה-Token, פתרון בעיות קריסה (Error Handling), שיפור זמני טעינה, ליטוש ה-CSS (Glassmorphism), ורספונסיביות. | 25 שעות |
| **9. העלאה לאוויר ותיעוד (Deployment & Docs)** | הגדרות וקינפוג של Vercel (ללקוח) ו-Render (לשרת), טיפול בבעיות CORS בייצור (Production), כתיבת קובץ README מלא, וכתיבת ספר/תיעוד הפרויקט להגשה. | 10 שעות |
| **סה"כ שעות עבודה** | **פרויקט גמר מקיף: Frontend + Backend + Database + Extension + AI** | **300 שעות** |

### טיפ להגשה:
אם ישאלו אותך איפה היה האתגר הכי גדול שלקח הכי הרבה זמן, תגיד שזה היה **סנכרון ההרשאות (Auth Sync) בין האתר (Web) לבין התוסף (Extension)** מבלי לסכן את ה-API Key, ושילוב של מודל ראייה ממוחשבת (Vision) דרך השרת אל מול גוגל. אלו משימות של מתכנתים סופר-מנוסים.
